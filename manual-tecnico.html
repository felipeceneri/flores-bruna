1. Arquitetura geral

O EstoqueApp é uma aplicação 100% client-side — sem servidor de aplicação próprio. Todo o processamento acontece no navegador do usuário, e os dados são persistidos no Supabase (PostgreSQL gerenciado).

Tabela usuarios — estrutura detalhada

-- Criação da tabela
CREATE TABLE usuarios (
  id          UUID     DEFAULT gen_random_uuid() PRIMARY KEY,
  nome        TEXT     NOT NULL UNIQUE,
  senha_hash  TEXT     NOT NULL,          -- SHA-256 hex
  ativo       BOOLEAN  NOT NULL DEFAULT true,
  permissoes  JSONB    NOT NULL DEFAULT '[]'::jsonb, -- array de strings
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

O campo permissoes é um array JSON com os valores: "produtos", "clientes", "lancamento", "pedidos", "relatorios", "usuarios".

Row Level Security (RLS)

A tabela usuarios tem RLS ativado com política permissiva — o controle de acesso é feito na camada de aplicação (JavaScript). A chave pública (publishable key) só permite as operações definidas pelas políticas.

Atenção: Para aumentar a segurança em produção, crie políticas RLS mais restritivas no Supabase limitando INSERT/UPDATE/DELETE apenas a usuários autenticados via Supabase Auth.
4. Sistema de autenticação

A autenticação é implementada sem dependência de JWT ou sessão server-side. O fluxo é:

  1. Usuário envia login + senha no login.html.
  2. A senha é hasheada com SHA-256 via Web Crypto API (crypto.subtle.digest).
  3. O hash é comparado com o senha_hash na tabela usuarios no Supabase.
  4. Se válido e ativo, os dados do usuário são gravados no sessionStorage.
  5. Ao fechar a aba, a sessão é destruída automaticamente.

auth.js — funções públicas

FunçãoDescrição
checkAuth(perm?)Verifica se há sessão. Se perm for informado, valida a permissão. Redireciona para login.html se não autorizado.
getSession()Retorna o objeto de sessão { id, nome, permissoes } ou null.
setSession(data)Grava os dados de sessão no sessionStorage.
logout()Remove a sessão e redireciona para login.html.

Exemplo de uso em uma nova página

<!-- No <head> da página -->
<script src="auth.js"></script>
<script>checkAuth('minha_permissao');</script>

Os itens do nav com data-perm="X" são ocultados via CSS (display:none) pelo auth.js quando o usuário não possui a permissão correspondente.

Importante: A ocultação no nav é visual. A proteção real acontece no checkAuth() chamado no <head> de cada página — o redirecionamento ocorre antes da página renderizar.

7. Como adicionar uma nova página
  1. Crie o arquivo nova-pagina.html copiando a estrutura de clientes.html.
  2. No <head>, adicione:
    <script src="auth.js"></script>
<script>checkAuth('nova_permissao');</script>
  3. Adicione "nova_permissao" ao campo permissoes dos usuários que devem ter acesso — via página de Usuários ou SQL direto.
  4. Adicione o item no nav de todas as páginas com data-perm="nova_permissao".
  5. Adicione a permissão aos checkboxes na página usuarios.html (seção #permGrid) e ao objeto PERM_LABELS.
8. Backup e manutenção

Exportar dados do Supabase

No painel do Supabase, acesse Table Editor → selecione a tabela → clique em Export (CSV ou JSON). Para backup completo, use o SQL Editor:

COPY (SELECT * FROM usuarios)   TO STDOUT WITH CSV HEADER;
COPY (SELECT * FROM SA1)        TO STDOUT WITH CSV HEADER;
COPY (SELECT * FROM SB2)        TO STDOUT WITH CSV HEADER;
COPY (SELECT * FROM SC5)        TO STDOUT WITH CSV HEADER;
COPY (SELECT * FROM SC6)        TO STDOUT WITH CSV HEADER;
COPY (SELECT * FROM produtos)   TO STDOUT WITH CSV HEADER;

Monitoramento

No Supabase, acesse Logs para ver queries em tempo real e detectar erros de acesso.

9. Redefinir senha de emergência

Caso todos os admins percam acesso, siga os passos:

  1. Abra o console do navegador (F12) em qualquer página e execute:
    crypto.subtle.digest('SHA-256', new TextEncoder().encode('NOVA_SENHA'))
  .then(b => console.log(Array.from(new Uint8Array(b))
    .map(x => x.toString(16).padStart(2,'0')).join('')))
  2. Copie o hash gerado (string hexadecimal de 64 caracteres).
  3. No Supabase → SQL Editor, execute:
    UPDATE usuarios
SET senha_hash = 'HASH_COPIADO_AQUI', ativo = true
WHERE nome = 'NOME_DO_USUARIO';
